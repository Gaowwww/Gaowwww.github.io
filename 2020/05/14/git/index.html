git

安装版本库

跟linux一样首先创建一个文件夹，然后进入文件夹中，然后git init，去初始化这个文件夹，然后这个文件夹就是git的仓库

git status

这个命令是看git的状态，如果修改了没有提交会由个提示没有提交的什么文件没有提交等信息

git diff

这个命令可以查看你与上次提交内容的对比

git add 文件名字

如果是新创建的文件名字，首先应该 例如：git add readme.txt，将文件加入到gitadd里面，目的使告诉git，将文件添加到仓库

git commit -m “文件说明”

将文件加入到git里面，然后再通过这样将文件提交到仓库。

git log

命令使用来查看日志，都什么时候更改了什么东西，可以加上–pertty=oneline参数

git reset –hard HEAD^

这个命令的作用使回到上个版本，就是上次修改的时候的内容，然后如果回到上上个版本就是^^

其实就是指针，然后通过指针访问不同的位置，还原不同的内容信息。

也可以是git reset –hard 加commit号就可以查看到了

git reflog

通过这个命令可以查看命令历史

git checkout – file

可以丢弃工作区的修改，有两种情况1.自修改后还没有被放到暂存区，现在，撤销修改就回到和版本一摸一样的状态；

2.添加到了暂存区后，又做了修改，现在撤销修改就回到暂存区，又做了修改，现在，撤销修改就回到添加到暂存区后的状态。

git rm

删除文件，git rm删除文件，如果误删除了，可以使用git chekout – 文件名 去回复，因为版本库里面还有这个文件。

分支管理。

通过双指针，一个指针指向新的东西，一个指向旧得东西，通过合并指针就会可以让指针完成重叠，完成版本库的更新。

创建一个分支

git checkout -b dev 去创建一个名字叫做dev的分支。这条命令相当于两条命令，一个是git branch dev 另一个 git checkout dev （这个命令是将当前的指针变成dev，就是管理dev分支）

查看当前的分支

git branch 分支前面会有个*来表示分支，可以在分支上面进行正常的提交等东西，

合并当前的分支

git merge，用于合并指定的分支到当前的分支，就是可以使用fgit merge dev 将dev分支合并到master中，

删除分支

git branck -d dev 可以使用这条命令去删除掉dev分支。

切换分支

可以使用git checkout +分支名字 去切换分支，也可以使用switch命令去切换

就是使用git switch -c dev 创建并且切换到新的dev分支，切换到已有的master分支，可以使用git switch master命令。

解决冲突

在分支合并的时候，有可能会产生冲突，所以就要解决冲突。

冲突，比如首先在分支上面更改一个文件，然后再主指针更改文件，分别提交。然后再合并就会产生冲突，因为两个文件都发生了更改都与之前的不一样，所以就会产生冲突，如果要解决的话，就在主指针重新进行一下add和commit，让其都再指向一个东西。这样两个发生改变的东西就会重叠到一个上面。

分支管理策略

通常合并分支时候，git会用fast forward模式，这种模式下删除分支之后，会丢掉分支信息。如果

合并分支的时候使用–no-ff 参数表示金用fast forward模式；具体就是

git merge –no-ff -m “merge with no-ff” dev

使用git log可以查看历史 使用 git log –graph –pretty=oneline –abbrev-commit

BUG分支

如果要修复一个bug分支，就要通过一个新的临时分支，来修复，修复后，合并分支，然后将临时的分支删除。这样就完成了修复，但是如果没有完成任务的时候，就有个功能，stash，可以把当前工作现场储藏起来，等以后恢复工程后继续工作。

使用git stash命令将工作区变成没有修改的时候，然后创建一个新的分支去修复bug，然后完成，然后去完成任务，因为git把stash内容存在某个地方了，但是要修复一下又两个办法：1.使用git stash apply恢复，stash内容并不删除，你需要用git stash drop来删除；2.是使用git stash pop删除，同时把stash你内容也删了。

可以使用git stash list 查看是否有stash内容

可以恢复指定git stash list 查看，并且使用git stash apply stash@【0】恢复指定的东西。

如果是再别的分支上面去，然后再继续修复master BUG，但是在由于哪个别的分支是在之前弄得，所以也会有bug，所以要把修复的bug弄到新的分支。

使用 git cherry-pick命令，就能指定修复。

后面的编号是commit编号。就是修复的bug的提交的哪个编号。

如果要添加一个新的功能最好在一个新的分支去更改，容易进行删除或者什么别的操作。

标签

是为了更好的管理，因为原来有commit编号，但是不好记，可以打上tag值然后去管理。

创建标签

git tag +名字 就在这个分支打上了标签。

使用 git tag 命令可以查看所有标签。

默认标签打在最新提交的commit上的，

标签不是按照事件顺序列出的，而是按照字谜要顺序排列的，可以用 git show 标签名字

去查看标签的信息。

git tag -a 指定标签名 -m 指定说明文字

操作标签

删除标签

git push origin v0.1 推送某个标签到远程。或者一次性推送所有标签到远程git push origin –tags
